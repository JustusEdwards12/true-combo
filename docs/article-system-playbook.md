# TrueCombo Article System Playbook

Use this playbook to build scalable, premium coaching lessons without one-off layouts.

## Required article sections

Major guides should include:

1. Intro summary (problem + win condition)
2. Key takeaways (3-5 bullets)
3. Practical scenarios
4. Common mistakes
5. Adaptation tips
6. Drill section
7. FAQ (when useful)
8. Progression links

## Frontmatter fields (optional but recommended)

```yaml
progressionTrack: neutral-core
progressionStep: 2
nextGuideSlug: how-to-whiff-punish
keyTakeaways:
  - Hold burst range first, then punish commitment.
  - Force one panic habit before hard reads.
faqs:
  - question: Should I always challenge recovery offstage?
    answer: No. Preserve stage if direct challenge is low-value.
relatedConcepts:
  - burst-range
  - whiff-punishing
```

## Progression design rule

Use progression chains rather than isolated links.

Example:

`What Is Neutral -> Burst Range -> Whiff Punishing -> Stage Control -> Conditioning`

## Linking standard

- Link to one prerequisite concept.
- Link to one practical follow-up guide.
- Link to one matchup or character application page.

Avoid generic labels like “Related guide”. Use specific labels like “Burst range glossary” or “Mario vs Samus matchup”.
